# AI Context & Coding Guidelines for Jatin-Gurwani.github.io

Welcome! This document provides essential project context, architectural guidelines, styling standards, and workflows for AI assistants collaborating on this Astro-based portfolio website.

---

## 🛠️ Tech Stack & Integrations

- **Framework**: [Astro v6](https://astro.build/) (Static Site Generation)
- **Styling**: [UnoCSS](https://unocss.dev/) with `presetWind3` (Tailwind v3 compatibility) and `presetIcons`.
- **Language**: TypeScript (strict configuration extends `astro/tsconfigs/strict`).
- **Integrations**:
  - `@astrojs/sitemap` (automatically generates `sitemap-index.xml` on build).
- **Deployment**: Automatic deployment to GitHub Pages via GitHub Actions on push to `main` branch.

## 📐 Architecture & Coding Guidelines

### 1. Data-Driven Components (CRITICAL)
- **Do NOT hardcode personal details or content** (like experience descriptions, contact emails, social URLs, project lists, or titles) inside Astro components.
- All dynamic and personal information must be read from the central configuration file: [src/config/portfolio.ts](file:///d:/git_projects/Jatin-Gurwani.github.io/src/config/portfolio.ts).
- If you need to expose a new visual attribute or text block, first define its type and add it to `portfolio.ts`, then reference it in the component.

### 2. Styling Rules (UnoCSS)
- This project uses **UnoCSS** for maximum performance and utility-first styling.
- **Shortcuts**: Use configured shortcuts in [uno.config.ts](file:///d:/git_projects/Jatin-Gurwani.github.io/uno.config.ts) to maintain design consistency:
  - **Cards**: `glass-card`, `glass-card-hover`
  - **Buttons**: `btn-primary`, `btn-secondary`, `btn-ghost`
  - **Typography**: `text-gradient`, `heading-xl`, `heading-lg`, `heading-md`, `body-lg`, `body-md`
  - **Containers**: `section-container`, `section-padding`
  - **Badges**: `tag`, `tag-sm`
- **Dynamic Themes**: The theme utilizes custom CSS variables generated by `src/utils/colors.ts` and set in `BaseLayout.astro`. Colors like `primary`, `accent`, and `surface` map automatically to light/dark themes.

### 3. Path Aliases
Always use the configured TypeScript path aliases when importing:
- `@components/*` points to `src/components/*`
- `@config/*` points to `src/config/*`
- `@layouts/*` points to `src/layouts/*`
- `@styles/*` points to `src/styles/*`
- `@scripts/*` points to `src/scripts/*`

### 4. Headings & SEO
- Maintain a single `<h1>` tag per page (primarily inside `Hero.astro`).
- Ensure SEO tags in `BaseLayout.astro` are kept dynamic and populated using properties passed through Astro layout parameters or fallback configurations from `portfolio.ts`.
- Ensure JSON-LD Structured Data (Person schema) in `BaseLayout.astro` is maintained.
